## 一、GAS 後端設置(老師端,只需做一次)

### 1. 進到你的 Apps Script 專案

打開你給我的那個 Sheet,點 **擴充功能(Extensions) → Apps Script**。
編輯器左邊應該已經有一個空的 `Code.gs`(或 `程式碼.gs`),旁邊有 ⊕ 按鈕可以新增檔案。

### 2. 貼進兩支 .gs 檔案

| 我寫好的檔案(在 repo 裡) | 貼到 Apps Script 的位置 |
|---|---|
| [`gas/Code.gs`](gas/Code.gs) | 取代原本空白的 `Code.gs`(整份內容貼進去) |
| [`gas/sheet_setup.gs`](gas/sheet_setup.gs) | 點左邊 ⊕ → **指令碼(Script)** → 命名 `sheet_setup` → 貼進去 |

貼完左邊應該看到兩個 .gs 檔案。**點存檔(磁片圖示)**。

### 3. 建立四張分頁

在編輯器上方下拉式選單選 **`setupSheets`**,點 **▶ 執行**。
- 第一次會跳出**授權**對話框 → 選你的 Workspace 帳號 → 「進階」 → 「前往(不安全)」 → 允許
- 跑完應該看到 alert:**「✅ 四張分頁已建立完成!」**
- 切回 Sheet 確認,左下應該有 4 個分頁:`students` / `conversations` / `game_state` / `species_prompts`,並且 `species_prompts` 第二列已經有一筆 `generic`

> 如果跳出錯誤「找不到分頁:xxx」表示授權沒給對 spreadsheet,重做一次授權即可。

### 3.5 在 students 分頁填入學生資料 ⚠️ 必填

登入畫面會比對 `students` 分頁的 `name` + `player_code`,**沒填學生就沒人能登入**。

開 Sheet 的 `students` 分頁,從第 2 列開始填:

| name | player_code | species | team |
|---|---|---|---|
| 王大名 | WD-7842 | generic | A 隊 |
| 林小美 | LM-3251 | generic | B 隊 |
| ... | ... | ... | ... |

欄位說明:
- **name**:學生姓名,登入比對時 trim 後**完全相符**才過(全形空白會比對失敗,小心)
- **player_code**:玩家代號,自動轉大寫比對,班內唯一
- **species**:GAS 端 chat 用來找對應 prompt 的 key。先全填 `generic`(對應 species_prompts 第二列那筆),AI 會用通用人格回應。日後想讓不同學生扮演不同物種,在 `species_prompts` 加新列(例如 `red`/`green` 或 `紅火蟻`/`石虎`),這欄填對應名稱即可
- **team**:目前前端不用,純記錄方便老師分隊

> 測試用最簡單方式:先加一列 `TEST | TEST | generic | -`,登入頁就用 TEST/TEST 試,通過代表整條鏈打通了。

### 4. 設定 Gemma API Key

在編輯器上方下拉式選單選 **`setApiKey`**,點 **▶ 執行**。
- 跳出對話框,把你剛剛的 `AIzaSy...` 貼進去 → 確定
- 看到 **「✅ Gemma API Key 已儲存」** 就好了

### 5. 測試 Gemma 是否通

選 **`testGemmaApi`** → ▶ 執行。
打開下方「執行記錄(View → Logs)」,應該看到:
```
✅ API Key 已設定,前 10 碼:AIzaSyXXX...
HTTP 狀態碼:200
✅ AI 回應:你好!很高興認識你。
Token:{"promptTokenCount":xxx, ...}
```
若 HTTP 不是 200,常見問題:
- 403 → API Key 沒在 Google Cloud 啟用「Generative Language API」
- **404 模型不存在** → 跑 `listAvailableModels` 看你的 key 可用哪些模型,把 `Code.gs` 第 33 行的 `GEMMA_MODEL` 改成清單裡的有效名稱(推薦 `gemini-2.0-flash`,最穩),存檔後重新部署
- 429 → 配額爆了,等一下再試

### 6. (選擇性)測試完整 chat 流程

選 **`testChatFlow`** → ▶ 執行 → 看執行記錄,應該回 `{"ok":true,"reply":"..."}`。
這時切到 Sheet 的 `conversations` 分頁應該看到兩筆新紀錄(user + assistant)。

### 7. 重新部署 Web App(重要!)

GAS 的部署是**版本快照**,你貼新 code 不會自動上線,**要手動建立新部署**:

1. 編輯器右上角點 **部署 → 管理部署**
2. 找到你已經給我的那個部署(URL 結尾是 `...VEDUHCFvA8/exec` 那個)
3. 點該部署右邊的 ✏️ 編輯圖示
4. **版本** 下拉選 **新版本**
5. 描述隨便寫(如 `Phase A 上線`)
6. 點 **部署** → 確定
7. **URL 不會變**(這是同一個部署的新版本),前端的 `config.js` 不用改

> ⚠️ **千萬不要點「新增部署」**,那會給你一條新 URL,前端就連不到了。

### 8. 用 api-test 確認連得上

如果你已經在本機(或 GitHub Pages 上),把 `src/api-test.html` 開起來:
- 上方應該顯示你的 GAS_URL 和 `MOCK_MODE: false`
- 按 **「健康檢查 (GET)」** → 應該看到綠字 `{ok: true, service: "habitat-dashboard-gas", ...}`
- 按 **「送 chat」** → 應該看到 `{ok: true, reply: "..."}`,旁邊顯示耗時(通常 1500–3500 ms)

✅ 後端就好了。

### 2. 啟用 Pages 並指定 src/ 為網站根

GitHub Pages 預設把 repo 根目錄當網站根目錄。我們的 HTML 在 `src/`,有兩個方式處理:

**方案甲(簡單,推薦):用 docs 資料夾或 main 分支根**

**方案乙(乾淨):用 GitHub Actions 把 src/ 部署成網站根**

#### 方案甲 — 直接用 main 分支,網站根 = repo 根

需要把 `src/` 內容**搬到 repo 根目錄**(或建立一個簡單的 `index.html` 重導),改動:

```powershell
# 把 src/ 裡的所有檔案搬到 repo 根
cd c:\NCU\RuRu\game0506
Move-Item -Force src\* .
Remove-Item src

# 重新 commit
git add -A
git commit -m "Move src/ to repo root for Pages"
git push
```

完成後:
1. 開 https://github.com/yuyu0216/species_v2/settings/pages
2. **Source**: Deploy from a branch
3. **Branch**: `main` / `/ (root)`
4. 點 **Save**
5. 等 1–2 分鐘 → 上方會顯示 `Your site is live at https://yuyu0216.github.io/species_v2/`

> 缺點:repo 根目錄會變得有點亂(html、js、css、png 全混在一起)。

#### 方案乙 — 用 docs/ 資料夾(維持 src/ 結構整齊)

```powershell
cd c:\NCU\RuRu\game0506
# 把 src/ 重新命名為 docs/(GitHub Pages 支援)
Move-Item src docs-site
# 注意:repo 已有 docs/(PRD.md 在那),改用別的名字
```

或者最簡單:**把 `src/` rename 成 `docs/`**,但你已經有 `docs/PRD.md` 了會衝突。

→ **我建議方案甲**(直接搬到根),如果你有顧慮告訴我我幫你寫 GitHub Actions。

## 四、MOCK 模式(展示用)

如果課堂上突然網路不穩或老師要 demo 但不想真的打 Gemma:

1. 打開 `src/js/config.js`
2. 改 `MOCK_MODE: true`
3. 推 commit + 等 Pages 重新部署(1–2 分鐘)
4. 重新整理學生瀏覽器 → 頂部會出現橘色 `[MOCK]` 標籤

6. 用完記得改回 `false` 再 push

> 課堂用法建議:課前先用 `MOCK_MODE: true` 走一遍流程確定 UI 沒問題 → 改 false → push → 開課。

## 七、故障排除

| 症狀 | 原因 | 解法 |
|---|---|---|
| 對話送不出去,顯示「我好累…」 | GAS 端錯誤 / 網路斷 / Gemma 配額爆 | 開 Apps Script → 執行記錄 看錯誤;或開 `api-test.html` 跑健康檢查 |
| `api-test.html` 健康檢查回 HTTP 401 / 403 | GAS 部署的存取權限不是 Anyone | 部署 → 管理部署 → 編輯 → Access: Anyone(沒帳號也可) |
| 回應是 HTML 不是 JSON | URL 沒填對(可能少了 `/exec`) | 檢查 `config.js` 的 GAS_URL 結尾必須是 `/exec` |
| 修改 GAS code 後對話沒變 | 沒重新部署 | 部署 → 管理部署 → 編輯 → 版本 → 新版本 → 部署 |
| `setupSheets` 報錯「找不到分頁」 | 授權給錯 spreadsheet | 重新執行,授權選對的帳號與 spreadsheet |
| 學生瀏覽器頂部突然出現 `[MOCK]` | `config.js` 的 `MOCK_MODE` 被誤改成 true | 改回 false,push,等 Pages 重新部署 |
| 同名同代號學生的對話 mix 在一起 | speciesAI 的設計,以 (name, player_code) 為對話 key | 老師發代號時確保唯一性 |
| 登入按鈕一直顯示「驗證中...」沒反應 | GAS 沒回應(網路 / Gemma 慢) | 等 15 秒會 timeout 跳錯誤;檢查 GAS 部署狀態 |
| 登入回「找不到此學生,請確認姓名與代號」 | students 分頁沒對應列、姓名/代號拼錯、或姓名前後有空白 | 開 sheet 比對 — name 要**完全相符**(含全形/半形空白) |
| 登入回「網路連線異常」之類 | GAS_URL 錯 / GAS 沒重新部署 / 部署存取權不是 Anyone | 開 `api-test.html` 跑健康檢查確認連得到 GAS |
| 倒數歸零後沒事發生 | 設計如此,Phase A 不自動結算 | 老師手動推進(Phase B 會處理) |
| Pages 部署後還是 404 | DNS / 快取 | 等 5 分鐘,或開無痕模式 |
